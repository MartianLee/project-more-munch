# More Munch 식당 수집 전략

## 현재 문제

카카오 카테고리 검색 API는 **한 좌표 기준 최대 45개** (3페이지 x 15개)만 반환하며, 이 제한은 API 자체 한도로 확장 불가 (카카오 공식 확인). 현재 구현은 사용자 위치 중심으로 1회 검색하므로 항상 45개에 고정됨.

## API 제약 조건

### 카카오 Local API (주 수집원)
| 항목 | 제한 |
|------|------|
| 페이지 | 최대 45페이지 (page 1~45) |
| 페이지당 결과 | 최대 15개 (size 1~15) |
| **조회 가능 결과** | **최대 45개/요청** (pageable_count 상한) |
| 검색 반경 | 최대 20,000m |
| 일일 호출 | 100,000회 (무료) |
| 월간 호출 | 3,000,000회 (무료) |

### 네이버 Local Search API (보조 수집원)
| 항목 | 제한 |
|------|------|
| 페이지당 결과 | **최대 5개** (display, 트래픽 정책으로 축소됨) |
| 최대 오프셋 | 1,000 |
| 일일 호출 | 25,000회 |
| 검색 방식 | 텍스트 쿼리 기반 (좌표 검색 불가) |

## 핵심 전략: H3 그리드 기반 분할 검색

카카오 API의 45개 제한을 우회하는 업계 표준 방법: **검색 영역을 작은 셀로 분할**하여 각 셀마다 독립 검색.

### H3 Hexagonal Grid

Uber가 만든 지구 전체를 정육각형으로 분할하는 체계. 정사각형 그리드 대비 장점:
- 모든 인접 셀까지의 거리가 동일 (정사각형은 대각선이 더 김)
- 검색 반경과 셀 크기가 자연스럽게 매칭

| 해상도 | 셀 면적 | 변 길이 | 용도 |
|--------|---------|---------|------|
| 7 | 5.16 km² | 1.41 km | 도시 구역 단위 |
| **8** | **0.74 km²** | **531 m** | **일반 도시 식당 검색 (추천)** |
| 9 | 0.11 km² | 200 m | 강남/홍대 등 밀집 지역 |

### 적응형 세분화 (Adaptive Refinement)

1. 해상도 8로 시작 (반경 ~600m)
2. 검색 결과가 45개(포화)이면 → 해당 셀을 해상도 9로 세분화 (~7개 자식 셀)
3. 포화되지 않은 셀은 그대로 유지

이 방식으로 밀집 지역만 세밀하게, 한산한 지역은 효율적으로 검색.

### 예상 수집량

사용자 위치 기준 도보 10분 (800m) 영역:
- 해상도 8 셀 약 **3~5개** 커버
- 셀당 최대 45개 → **135~225개** 식당 수집 가능
- API 호출: 셀당 3페이지 = **9~15회** (일일 한도 100,000회의 0.015%)

도보 15분 (1,200m) 영역:
- 해상도 8 셀 약 **7~10개**
- **315~450개** 식당 수집 가능
- API 호출: **21~30회**

## 수집 주기 전략

### 초기 수집 (Bootstrap)
- 사용자 설정 완료 시 즉시 실행
- 주변 영역 전체 그리드 스캔
- 1회성, 수 초~수십 초 소요

### 일일 갱신 (Daily Delta)
- 매일 새벽 4시 크론
- 전체 셀을 7그룹으로 나눠 1그룹/일 순환 → 주간 전체 리프레시
- 호출량 최소화

### 주간 전체 스캔 (Weekly Full)
- 일요일 새벽 4시
- 모든 셀 재검색
- 7일 이상 미갱신 식당 비활성화

## Rate Limit 관리

```
요청 간격: 100ms (초당 10회)
429 응답 시: 지수 백오프 (1s → 2s → 4s, 최대 30s)
Retry-After 헤더 존중
일일 한도 80% 도달 시 수집 중단
```

## 네이버 API 활용 전략

네이버는 좌표 기반 검색이 불가하므로 **보조 수집원**으로만 활용:

1. 카카오에서 수집된 식당 이름으로 네이버 검색 → 평점/리뷰 수 보강
2. "시청역 맛집", "을지로 점심" 등 **지역명 + 키워드** 조합으로 추가 식당 발견
3. `display=5` 제한이므로, 쿼리를 다양화하여 결과 다양성 확보

## 구현 계획

### Phase 1 (즉시): 단순 그리드 확장
- 사용자 위치 중심으로 해상도 8 H3 셀 목록 생성
- 각 셀 중심점으로 카카오 검색 (반경 600m)
- `kakaoId`로 중복 제거
- 요청 간 100ms 간격

### Phase 2 (선택): 적응형 세분화
- 포화 셀 감지 및 해상도 9 세분화
- 수집 통계 로깅 (셀별 결과 수, 포화율)

### Phase 3 (선택): 네이버 교차 보강
- 카카오 수집 후 네이버로 평점/리뷰 수 보강
- 네이버 전용 키워드 검색으로 누락 식당 추가
